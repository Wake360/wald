# M2 plan — LLM narrative layer (2026-07-04)

Synthesis of a 3-designer + 2-adversarial-critic workflow run over the
master plan (`LifeOS/outputs/wald-plan.md` §4.5–4.6, Přílohy A/C/K) and the
current repo. Design decisions below record where the critics overruled
the designers and why.

Deliverable: layer B (narrative detector) + verifier + fusion, gated by
G2 (per-class F1 ≥ 0.7 on narrative classes, clean FP ≤ 10%, 100% flags
carry two evidence spans) and G3 (verifier kills ≥ 80% of seeded false
flags). Termination rule unchanged: two prompt/fusion iterations, then
static-only v1 if recall ≥ 0.6 with FP ≤ 10% cannot hold.

## Phase split

- **Phase 1 (now, $0):** everything below except the runs — mutations,
  negative corpus, held-out split, backend seam, detector, verifier,
  fusion, eval extension, all tested against replay fixtures.
- **Phase 2 (still $0):** prompt iteration via an agent backend
  (`claude -p`) on the dev split only. De-risking, never gate evidence.
- **Phase 3 (blocked):** needs TWO keys — Anthropic (detector) and
  OpenAI (verifier; cross-provider is a hard constraint). Projected
  spend ~$30–45 of the $150 budget (ledger below).

## Scope decisions (from the critique round)

1. Only the three mutant-backed classes enter the prompt:
   `significance-meaningless`, `regression-to-mean-claim`,
   `selection-survivorship-cohort` (fused). `linearity-extrapolation`
   and `leakage-future-feature` stay out of M2 entirely — a class with
   no mutants cannot be gate-measured, and an unmeasured class in a
   closed-taxonomy prompt is pure FP surface. Marked
   `narrative_enabled: false` in flaws.yaml.
2. No PreCVFitMutation, no new corpus family. The pre-CV fusion rule
   ships but its recall is sanity-checked against the 4 real contested
   candidates from the dogfood set, reported as its own bucket outside
   G2 F1. Its FPs on clean notebooks DO count toward G2's FP ≤ 10%
   (FP protection is universal; recall claims only where mutants exist).
3. Confidences are fixed per rule, never the model's number:
   fused survivorship 0.91, fused pre-CV 0.88, solo narrative
   verifier-supported 0.80 (all inside the Příloha K bands; 0.80 sits
   at the default floor — verifier support is the promotion mechanism,
   no second floor). `model_confidence` goes to `Flag.extra` only.
4. Calibration/reliability diagram is parked to M3 with an explicit
   statement: at ~30 held-out flags the bins cannot distinguish 0.75
   from 0.95 precision. Bin counts are reported, calibration is not
   claimed.
5. No budget-enforcement machinery, no confirmation channel, no
   grounding-repair heuristics, no pipeline.py. Cut as single-consumer
   abstractions; the fallbacks are: token totals in the report,
   post-hoc fusion joins, verbatim-match-or-drop, `run_full` in fuse.py.

## Work items (Phase 1, ordered)

1. **flaws.yaml**: add `narrative_enabled` and `disqualifiers` (the
   "this is NOT the flaw" list per class, lifted from Příloha M — e.g.
   survivorship: claim explicitly scoped to the filtered group). The
   disqualifiers feed both the detector and verifier prompts; they are
   the FP control.
2. **mutate.py — SignificanceMeaninglessMutation** (abtest family):
   data-gen cell rewritten to n=20 000/group with true effect ~0.04 SD
   (p<0.05 near-certain, d negligible); the Cohen's d computation STAYS
   in code (the flaw is the prose ignoring it — master plan A.5);
   conclusion from a seed-indexed pool of ≥4 phrasings, all citing p and
   issuing a rollout recommendation, none mentioning effect size.
   verify(): execution probe asserts p<0.05 and |d|<0.2; regex asserts
   no effect-size token in markdown, recommendation verb present.
3. **mutate.py — RegressionToMeanMutation** (churn family): the
   family's data-generation cell gains a second-period column generated
   with test-retest correlation r≈0.5 (ordinary synthetic data, not a
   giveaway cell inside the mutant); the mutation inserts extreme
   selection (`nsmallest`/quantile, seed-varied) + period comparison +
   causal conclusion from a ≥4-phrasing pool, no control group.
   verify(): AST — extreme selection present, no control-group
   construction; execution probe — selected group's T+1 mean moved
   toward the grand mean; regex — causal verb present, no control
   language.
4. **mutate.py — SurvivorshipMutation hardening**: replace the single
   POPULATION_MD constant with ≥4 phrasings × 2 filter idioms
   (subscript filter and `.query`); generalize verify() from fixed
   substring to population-quantifier regex + absence of scoping tokens.
5. **corpus.py — held-out split (risk R3)**: `split: dev|heldout` per
   MANIFEST entry. Dev = existing base seeds, mutation seeds 0–1,
   phrasing variants 0–1. Held-out = fresh base seeds, mutation seeds
   2–3, phrasing variants 2–3, PLUS one entire rhetorical family of
   conclusions (different sentence shape, not word swaps) that exists
   only in held-out. G0-extension test asserts three disjointnesses
   (base files, seeds, phrasing strings) from the MANIFEST.
6. **corpus.py — `build_negative_flags`** → `corpus/negative/`
   (G3 substrate). Five deterministic recipes, each with a mechanical
   falseness check:
   - `scoped-claim`: cite the clean cohort's genuinely scoped
     conclusion + filter line as survivorship. The scoping token sits
     in the CELL, not necessarily the quote — this is the case a
     span-only verifier cannot kill and the reason the verifier gets
     full cell text.
   - `effect-size-present`: p-value sentence from a notebook that also
     reports d/CI, flagged as significance-meaningless.
   - `control-group-present`: period comparison with an
     equally-extreme control constructed in the cited code, flagged as
     regression-to-mean.
   - `wrong-code-span`: real population-sounding claim + the imports
     cell (evidence-discipline test).
   - `legit-cv-generalization`: correct Pipeline-in-cross_val_score +
     a generalization claim, flagged as leakage — mined from the
     dominant idioms of the 119 real dogfood FPs ("pipeline used
     correctly", "CV manages splits internally"). This recipe is
     RESERVED ENTIRELY for held-out G3; recipes 1–4 split within-recipe.
   G3 is reported per recipe; aggregate-only kill rates are gameable by
   the two near-mechanical recipes.
7. **llm.py — backend seam** (~4 classes, one method):
   `Backend.complete(system, user, schema) -> dict` with attrs
   `provider`, `model` (pinned id), `kind` in {api, replay, agent}.
   `AnthropicBackend`, `OpenAIBackend` (thin, retry-once-on-bad-JSON,
   raise at call time if key missing); `ReplayBackend(dir, inner=None)`
   — sha256(system+user)-keyed record/replay, raises on miss with the
   hash (this one class covers mock fixtures, caching, and free
   re-runs); `AgentBackend` — shells out to `claude -p`, kind=agent.
   Every response carries its kind; `gate_evidence` is COMPUTED (all
   calls kind==api, models==pinned) — never asserted.
8. **narrative.py — layer B detector**: `package_notebook` (all cells
   in order with true indices; markdown never truncated; outputs capped;
   ~12k-token budget with one truncation step, logged); system prompt
   generated from flaws.yaml (narrative_enabled classes only:
   definition + disqualifiers + failure_scenario; snapshot test pins
   it); single call per notebook returning
   `{claims: [{cell, quote, kind, code_cells}], findings: [{flaw_id,
   claim_span, code_span, failure_scenario, fix, model_confidence}]}`
   with kind a closed enum (population | causal |
   performance-generalization | forecast | significance | descriptive).
   Grounding is mechanical, verbatim-after-whitespace-normalization or
   drop: claim quote must be a substring of the cited markdown cell,
   code quote of the cited code cell (≤400 chars — no whole-cell
   gaming), line numbers recomputed from the match, flaw_id checked
   against the enabled set, malformed JSON retried once then empty.
   Drops counted as `dropped_ungrounded` — a gated metric, see G2 below.
9. **verifier.py**: verifier receives the flaw definition +
   disqualifiers + both quotes WITH the full text of their two cells
   (quotes alone are blind to a scoping sentence one line away).
   Prompt per Příloha C.2, skeptical framing. Output
   `{verdict: supported|unsupported, reason}`; fail-closed (unparseable
   = unsupported). Constructor asserts detector.provider !=
   verifier.provider (replay/test backends use distinct provider
   strings).
10. **fuse.py + taxonomy/fusion.yaml**: three declarative rules (the
    master plan mandates rules-as-data; semantics stay minimal — two
    join predicates only):
    - survivorship: static candidate (0.50–0.79) + narrative finding of
      the same flaw whose code_span cell == the candidate's cell →
      0.91, verifier required.
    - pre-CV: static `leakage-fit-before-split` candidate at 0.70–0.79
      + a performance-generalization claim whose `code_cells` include
      the CV sink cell (recorded by detect.py in `Flag.extra`) → 0.88,
      verifier required. (A kind-only join with no link predicate fires
      on every teaching notebook — critic-confirmed FP trap.)
    - solo narrative finding, verifier-supported → 0.80.
    `run_full(nb, det_backend, ver_backend)` = static → narrative →
    fuse → verify → drop unsupported. Fused rules consume their
    narrative finding (no double flag).
11. **eval.py**: extend `evaluate()` with a flags-producer parameter
    (default `run_static`; LLM path passes `run_full`) instead of a
    sibling reimplementation. Narrative extras: per-class P/R/F1 by
    split, `dropped_ungrounded` rate, G3 kill rate per recipe, true-flag
    survival, backend provenance + pinned models + floor + token/$
    totals, `gate_evidence`. STRUCTURAL guard: evaluating
    split=heldout with any non-gate-eligible backend raises — agent and
    replay runs cannot touch held-out at all, contamination is
    prevented, not labeled. Gate runs use a fresh replay dir; cached
    re-runs are never gate evidence.
12. **cli.py**: `wald check --llm`, `wald eval --llm --split dev`
    (heldout only via the gate path). Default stays static-only.
13. **tests**: fixture-driven end-to-end (clean, survivorship mutant,
    RTM mutant) reproducing the Příloha L trace shape; counter-test
    (scoped claim → no fusion, no flag); grounding drop cases; provider
    collision; heldout-raise; G0-extension disjointness.

## Gates (restated with the critics' fixes)

- **G2** (held-out split only, floor 0.8 pinned in the report):
  per-class F1 ≥ 0.7 for the three narrative classes; clean FP ≤ 10%
  over held-out synthetic clean + corpus/real (caveat printed in the
  report: corpus/real shaped the static layer during dogfood, so the
  fused-FP number leans optimistic — it is still the only real-notebook
  FP evidence available); 100% of surviving flags carry two validated
  spans (holds by construction) AND `dropped_ungrounded` ≤ 10% of raw
  findings (so the construction cannot hide a model that fails
  grounding constantly).
- **G3** (held-out negatives only): kill ≥ 80% overall AND reported per
  recipe, including the never-tuned-on `legit-cv-generalization`
  recipe; true-flag survival ≥ 0.75 on the same runs (a
  kill-everything verifier must not pass).
- **R3 tripwire**: dev-vs-heldout recall gap > 0.15 on any class =
  corpus overfit alarm; the held-out number is the honest one.

## Iteration budget (explicit, from the termination rule)

Agent-backend prompt iteration on dev is unlimited but dev-only (the
heldout raise enforces this). Each agent-tuned change requires one API
dev-run checkpoint before further spend; those checkpoints count as the
two allowed iterations. Max two gate attempts on held-out. After
iteration 2 without dev-level recall ≥ 0.6 at FP ≤ 10%: stop, ship
static-only, keep the corpus and harness (they survive either outcome).

## Cost ledger (Phase 3)

Detector ~5–12k tokens in / ~0.8k out per notebook (~$0.04–0.05 at
Sonnet-class pricing); verifier ~2k in / 0.1k out per flag (~$0.005).
Fixture-recording smoke run ~$1; dev run ~$2–3 each (iteration 0 + 2
checkpoints ≈ $9); G3 runs ~$0.5 each; gate attempts ~$4 each × 2;
LLM dogfood over the 34 real notebooks ~$4. Total ≈ $30, contingency to
$45; a top-tier detector retry before termination fires still fits
(~5× detector cost). The binding constraint is the two-iteration rule,
not the $150.

## Honest limits (written into the eval report up front)

- Narrative mutants are template prose from one author. The
  whole-family phrasing holdout and the drift alarm detect memorization
  but cannot rule it out; real-notebook recall evidence stays limited to
  the dogfood LLM run, reported as a separate number, never as G2.
- G3 negatives are mostly authored strawmen; one recipe is mined from
  real dogfood FP idioms and held out entirely — the closest available
  proxy for verifier behavior on real false flags.
- Confidence values are design constants checked against Příloha K
  bands; empirical calibration waits for M3 corpus size.
