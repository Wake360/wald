# Dogfood report — 2026-07-04 (real notebooks, first contact)

First run of wald on real-world notebooks. This is the test the synthetic
eval could not provide, and it failed exactly the way risk R1 predicted:
false-positive flood. This report records the before, the root causes, the
fixes, and the after. Aggregates only; no notebooks are attributed to
authors here (license attribution lives in `corpus/real/MANIFEST.json`).

## Setup

- 34 notebooks pulled from 5 permissively licensed GitHub repositories
  (Apache-2.0 / MIT; classic tabular ML and statistics teaching material —
  the closest key-free substitute for Kaggle kernels).
- `wald check` over all 34; then every flag hand-reviewed: one reviewer
  per notebook judging each flag TP/FP with an idiom classification, plus
  miss-hunters over unflagged notebooks, plus an adversarial verifier that
  tried to refute every claimed TP and claimed miss. Reviewers were LLM
  agents, not domain-expert humans — treat the ground truth accordingly.

## Before (detector as of 2026-07-02)

118 confident flags on 17/34 files — a 50% file flag rate on notebooks that
are mostly textbook-correct. Review verdicts on all 124 emitted flags
(confident + candidates): **3 TP, 119 FP, 2 candidate-level**. Even the 3
TPs survived only as contested (adversarial verifiers refuted them under a
strict reading of the flaw class). 13 cells failed to parse.

FP idioms (reviewer classification, n=124):

| idiom | n |
|---|---|
| fit on the train split only (correct usage) | 44 |
| fit on full data, no test evaluation (visualization/demo) | 24 |
| model fit for visualization | 21 |
| name reuse across unrelated notebook sections | 19 |
| other | 7 |
| cross-validation manages splits internally | 4 |
| true leakage | 3 |
| pipeline used correctly | 2 |

Root causes, all mechanical:

1. **Estimator/transformer conflation.** The detector flagged any
   `.fit`/`.fit_transform` consuming a split ancestor — including model
   fits (`SVC`, `KNeighborsClassifier`, `GridSearchCV`), which are not
   preprocessing leakage.
2. **Flow-insensitive dataflow.** Dependencies unioned across
   reassignments, so reusing `X` for a second dataset linked unrelated
   sections; `X_train` reused as a name anywhere poisoned the ancestor set.
3. **Parse fragility.** Indented magics broke block syntax when stripped;
   `%`-formatting continuation lines were misread as magics; `%%` cell
   magics with non-Python bodies produced parse errors.

## Confirmed misses (adversarially verified, 7 claims survived)

- Self-statistic imputation before the split
  (`data[c].replace(0, data[c].median())` then `train_test_split`) —
  2 notebooks. Pandas-level leakage, invisible to the sklearn-fit detector.
- Supervised feature selection on full data feeding cross-validation
  (`SelectKBest(chi2).fit(X, y)` → `cross_val_score`) — 1 notebook,
  the Elements-of-Statistical-Learning classic.
- Unsupervised transform on full data feeding CV (PCA) — same notebook.
- Per-class groupby mode/median imputation applied row-wise via `apply`
  before the split — 1 notebook, 2 sites.
- Target-correlation feature selection on full data inside a helper
  function — 1 notebook.

## Fixes (this commit)

- Leakage flags only transformer fits: `fit_transform` by definition,
  bare `.fit` only for known transformer classes or receivers later used
  with `.transform`. `LabelEncoder`/`LabelBinarizer` excluded (y-encoding
  is standard practice).
- The leak must reach an evaluation sink: the transformed output feeds
  `train_test_split`, or the full-data-fitted receiver transforms a split
  output.
- Flow-sensitive dependency chains with kill-on-reassign replace the
  global ancestor union.
- `cross_val_score`/`cross_val_predict`/`cross_validate` are evaluation
  sinks too: supervised transformer fit on the CV data → confident flag
  (0.9); unsupervised (scaler/PCA before CV) → candidate below the floor
  (0.75), because hand-review showed reviewers themselves disagree on it.
  Estimator arguments to CV are excluded (a pipeline passed to CV is
  cloned and refit per fold — correct usage).
- New pattern: `fillna`/`replace` with statistics of the same frame before
  the split (one flag per cell).
- Parsing: try raw Python first, preserve indentation when stripping
  magics, skip non-Python `%%` cell bodies silently.

## After (same 34 notebooks)

- **3 confident flags, all three = the reviewer-confirmed real leaks**
  (2× pre-split median imputation, 1× SelectKBest-before-CV).
- **0 known false positives at the confidence floor.** File flag rate
  3/34 (9%), all justified — under the ≤10% FP target.
- Contested pre-CV unsupervised fits appear as below-floor candidates
  (4 sites), which is where contested-by-design signals belong until the
  narrative layer (M2) can fuse them.
- Parse errors: 0 cells (was 13).
- Synthetic corpus gates unchanged: precision/recall 1.00 on all three
  static classes, 64/64 mutants still detected.

Known misses remaining (documented, deliberate): the groupby-`apply`
imputation idiom and function-scoped correlation selection have no
name-level syntactic anchor; they are candidates for M2/M3, not for more
static heuristics. Two further blind spots are the price of the
flow-sensitive rebuild and accepted for v1: a *conditional* rebind
(`if demo: X = load_small()`) kills the chain like an unconditional one,
and estimator-derived feature leakage (stacking a model's `predict`
output into features before the split) is out of scope now that estimator
fits are exempt.

## Corpus fold

27 of the 34 notebooks were reviewed clean and entered the corpus as
licensed real clean entries (`corpus/real/`, license per file in its
MANIFEST, image outputs stripped for size). 5 notebooks with confirmed
flaws and 2 duplicates were excluded. `wald eval` now measures the clean
FP rate over 47 notebooks (20 synthetic + 27 real): **0.0%**.

## Honest caveats

- Ground truth is LLM review with adversarial verification, not expert
  human labeling. The 3 TP / 119 FP split was unambiguous, but candidate-
  level judgments (pre-CV scaling) were contested even between agents.
- The sources are teaching-oriented repositories; production notebooks
  will have messier idioms. Recall on real flaws is measured against only
  7 confirmed instances — too few to report as a recall number.
